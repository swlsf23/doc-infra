# Long-form overview

::: note
Placeholder material for internal testing only. No operational, analytical, or evaluative use is intended.
:::

This section exists to stress-test the documentation shell when writers ship **multi-section chapters**, generous whitespace, and cross-references that span many screen heights. Nothing here is production guidance; it is deliberately verbose so you can scroll the main column, resize the viewport, and confirm that navigation, typography, and the sidebar remain usable.

## Why a dedicated long-form bucket

Real manuals rarely fit on a single viewport. Release notes, migration guides, and API references accumulate paragraphs that wrap differently on ultrawide monitors versus narrow laptops. By parking placeholder prose in one place, you can compare line length, heading rhythm, and list density without touching real content.

## How to use these pages

Treat each chapter as an independent scroll target. Toggle the sidebar, return home, and navigate back through the manifest-driven tree. If anything feels cramped or the iframe navigation misbehaves at depth, adjust `ux/site.css` or the site wrapper—not these stubs.

## Relationship to other UX lab pages

The **layouts** sibling folder explores wide tables and code blocks; **deep** adds another directory level under `ux-lab` so path-based URLs and relative links stay honest. Together they form a small matrix: length, width, and depth.
