# File locations (examples)

This repository uses the following layout for sample content:

- `content/src/getting-started/` — onboarding-style pages
- `content/src/guides/` — operational and author guidance
- `content/src/reference/` — lookups and definitions

Infrastructure files live outside `src/`, such as `content/manifest.yml` and `code/config.yml`.

## Note

Paths here describe **this** sample tree. Your real project can relocate `src/` as long as manifests and config agree.
